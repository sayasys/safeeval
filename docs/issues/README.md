# Framework-level friction items

Issues filed here are framework-level execution-friction items surfaced during normal SafeEval work that no single execution track is positioned to fix on its own. They are tagged for `tracks-architect` attention -- the architect decides whether each rises to a memo amendment, a workflow change, an upstream bug report (Cowork / OneDrive / Windows), or stays as a known limitation with a documented workaround.

Filename convention: `<short-kebab-slug>.md`. Each issue carries a date, a description of the failure pattern, the cost paid when it fires, the workaround currently in use, and a recommendation for the architect to consider.

This directory is committed (not gitignored) so the architect can see issues across multiple machines and sessions.
